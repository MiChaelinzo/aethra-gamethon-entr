## Essential Screenshots Needed

For a winning Devpost submission, you need 6-10 high-quality screenshots that showcase different aspects of your game. This guide tells you exactly what to capture.

## üéØ Screenshot Checklist

### 1. **Main Menu / Level Select** (REQUIRED)
**What to show:**
- Level selection screen with multiple biomes visible
- CO‚ÇÇ reduction total at the top
- Daily Challenge and Tournament buttons
- Difficulty mode selector (Normal/EXTREME)

**How to capture:**
- Open game in browser
- Wait for main menu to load
- Ensure no overlays are open
- Take screenshot at 1920x1080 or higher

**Why it matters:** First impression - shows scope and variety of game

---

### 2. **Active Gameplay with Particle Trails** (REQUIRED)
**What to show:**
- Game board with tiles clearly visible
- Active particle trail effects following cursor
- Score, moves, and pollution stats on right side
- Some tiles selected/highlighted

**How to capture:**
- Start any level (Forest recommended for good colors)
- Enable particle trails (`Ctrl + P` if disabled)
- Set intensity to High for dramatic effect
- Move mouse around board while taking screenshot
- Capture mid-game (not at start)

**Why it matters:** Shows core gameplay and unique particle system

---

### 3. **Educational Card Display** (REQUIRED)
**What to show:**
- Educational pop-up in foreground
- Game board visible but dimmed in background
- Clear text with renewable energy facts
- CO‚ÇÇ impact number visible

**How to capture:**
- Start Level 1
- Click a tile type you haven't matched before
- Immediately take screenshot when card appears
- Try Solar or Wind for recognizable icons

**Why it matters:** Demonstrates educational value and climate focus

---

### 4. **Power-Up Animation** (REQUIRED)
**What to show:**
- Large dramatic power-up effect on screen
- Supernova, Tsunami, or Phoenix activation
- Glowing matched tiles
- High score numbers visible

**How to capture:**
- Complete a daily challenge to unlock power-up
- Play level until you match the power-up tile
- Take screenshot during peak of animation (practice timing)
- May need several attempts to catch best moment

**Why it matters:** Shows advanced features and visual polish

---

### 5. **Daily Challenge Interface** (REQUIRED)
**What to show:**
- Daily challenge card with challenge details
- Streak counter showing days completed
- Reward power-up preview
- Time remaining until next challenge

**How to capture:**
- From main menu, click daily challenge button
- Wait for card animation to finish
- Ensure all text is readable
- Take screenshot before starting challenge

**Why it matters:** Highlights progression system and replayability

---

### 6. **Tournament Leaderboard** (REQUIRED)
**What to show:**
- Leaderboard with multiple player entries
- Player rankings (1st, 2nd, 3rd with medals)
- Scores and CO‚ÇÇ reduction columns
- Tournament name and description at top

**How to capture:**
- From main menu, click tournament button
- Let leaderboard fully load
- Ensure your entry is visible (play tournament first)
- Capture with at least 5-10 entries visible

**Why it matters:** Shows multiplayer/competitive features

---

### 7. **Badge Showcase** (RECOMMENDED)
**What to show:**
- Badge collection display
- Mix of earned and locked badges
- Player statistics panel
- Badge descriptions

**How to capture:**
- From main menu, click badge showcase button
- Scroll to show variety of badges
- Take multiple screenshots if needed
- Try to show at least 1-2 earned badges

**Why it matters:** Demonstrates achievement system

---

### 8. **Collision Heatmap Overlay** (RECOMMENDED)
**What to show:**
- Game board with heatmap overlay active
- Colored zones showing collision hotspots
- Heatmap controls visible
- Some zones glowing/colored

**How to capture:**
- Play several levels to build up collision data
- Press `Ctrl + H` to enable heatmap
- Continue playing to make some zones light up
- Take screenshot during active gameplay

**Why it matters:** Shows advanced analytics feature

---

### 9. **Level Complete Screen** (RECOMMENDED)
**What to show:**
- Victory screen with final score
- CO‚ÇÇ reduction for that level
- Star rating or completion badge
- "Next Level" and "Restart" buttons

**How to capture:**
- Complete any level successfully
- Wait for completion animation to finish
- Take screenshot of final screen
- Choose level with impressive stats

**Why it matters:** Shows win state and progression

---

### 10. **Tutorial/Help Overlay** (OPTIONAL)
**What to show:**
- Tutorial overlay with instructions
- Example gameplay shown underneath
- Step indicators (1/5, 2/5, etc.)
- Clean, readable text

**How to capture:**
- Clear browser data to trigger first-time tutorial
- Or click help button for tutorial
- Take screenshot of most informative step
- Ensure text is fully visible

**Why it matters:** Shows accessibility and user-friendly design

## üìê Technical Requirements

### Resolution
- **Minimum:** 1280x720 (HD)
- **Recommended:** 1920x1080 (Full HD)
- **Maximum:** 3840x2160 (4K) - may be overkill

### Format
- **PNG** preferred (best quality, no compression artifacts)
- **JPG** acceptable (use high quality setting, 90%+)
- Avoid GIF (unless showing animation, then use short video instead)

### File Size
- Devpost limit: Usually 5-10MB per image
- Optimize large files using TinyPNG or similar
- Don't over-compress - quality matters!

## üé® Pro Tips for Great Screenshots

### Composition
‚úÖ **DO:**
- Use full screen mode (F11) to remove browser UI
- Ensure all text is readable
- Show colorful, interesting game states
- Capture exciting moments (big combos, explosions)
- Include UI elements that explain what's happening

‚ùå **DON'T:**
- Show browser tabs, bookmarks, or desktop
- Capture loading screens or errors
- Include personal information (if any)
- Take screenshots of empty/boring states
- Leave debug overlays visible

### Timing
- **Practice:** Run through game flow before screenshotting
- **Animation:** For effects, take multiple shots and choose best
- **Loading:** Wait for everything to fully render
- **Hover States:** Show interactive elements in active state

### Variety
- Mix overview shots with detailed close-ups
- Show different biomes/levels for visual variety
- Include both menu screens and gameplay
- Alternate between calm and action-packed moments

## üñºÔ∏è Screenshot Workflow

### Recommended Process

1. **Plan Your Shots** (5 mins)
   - Read checklist above
   - Decide which 8-10 you'll capture
   - Note which require special setup

2. **Prepare Game State** (10 mins)
   - Complete daily challenge for power-ups
   - Play some levels to generate data
   - Enter tournament for leaderboard entries

3. **Capture Screenshots** (20 mins)
   - Work through checklist systematically
   - Take 2-3 of each type, choose best
   - Check each immediately for quality

4. **Post-Process** (10 mins)
   - Rename files descriptively
   - Resize if needed (maintain aspect ratio)
   - Optimize file sizes
   - Create backup folder

5. **Organize** (5 mins)
   - Number files in order you'll upload
   - Write caption for each
   - Test that all open correctly
